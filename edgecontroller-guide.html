1. What is EdgeController?

EdgeController is a self-hosted controller that runs on a Raspberry Pi 5. It provides a browser-based interface for managing connected BLE and USB devices, building automated macro sequences, and accessing your setup remotely over a secure Cloudflare tunnel.

There is no cloud subscription, no app to install, and no accounts to create. Everything runs on the Pi on your local network. Remote access is set up automatically the first time you connect to WiFi.

Key Features

  • Web interface — accessible from any browser on your network
  • BLE device management with automatic reconnection
  • Visual macro editor — build complex automated sequences with no coding
  • StreamDeck+ integration — hardware buttons and encoders for live control
  • Secure remote access via Cloudflare tunnel (web + SSH)
  • WiFi onboarding — guided setup via a built-in hotspot

3. Setting Up Your Device

Step 1 — Flash the SD card

  1. 1Open Raspberry Pi Imager on your computer.
  2. 2Click Choose OS and select Use custom image. Navigate to the edgecontroller-v2.0.img.gz file you were provided.
  3. 3Click Choose Storage and select your SD card.
  4. 4Click the settings icon (gear). You do not need to set a hostname, username, password, or WiFi — these are all preconfigured in the image. Skip all settings.
  5. 5Click Write and wait for it to complete. This takes 5–10 minutes.
Do not eject the card while writing is in progress. Doing so will corrupt the card and you will need to reflash it.

Step 2 — First boot

  1. 1Insert the SD card into the Pi and power it on.
  2. 2Wait approximately 60–90 seconds for the first-boot setup to complete. During this time the Pi assigns itself a unique device ID based on its hardware serial number.
  3. 3The Pi will broadcast a WiFi hotspot named EdgeController-XXXX (where XXXX is its device ID).

Step 3 — WiFi setup

  1. 1On your phone or computer, connect to the EdgeController-XXXX WiFi network. The password is edgesetup.
  2. 2Open any webpage in your browser. You will be automatically redirected to the EdgeController setup page.
  3. 3Enter your home or venue WiFi network name and password, then click Connect.
  4. 4The Pi will connect to your WiFi and restart. Reconnect your device to your normal WiFi.
  5. 5After 30–60 seconds, remote access is configured automatically. You will receive your personal device URL.
If you're connecting the Pi via Ethernet instead of WiFi, you can skip the hotspot setup. Access the interface directly on your local network using the device address shown below.

4. Connecting to the Interface

Local access (same network)

Once the Pi is connected to your WiFi, open a browser and go to:

http://XXXX.local:3000

Replace XXXX with your device ID (shown at the end of the install, or printed on your device if labelled). Example: http://c4a5.local:3000

.local addresses use mDNS (Bonjour). This works automatically on Mac and most phones. On Windows you may need to install Bonjour or use the Pi's IP address instead.

Remote access (from anywhere)

After your first WiFi connection, EdgeController automatically sets up a secure Cloudflare tunnel. Your personal remote URL is:

https://XXXX.kinkcontrol.org

This URL works from anywhere in the world with no VPN or port forwarding required. It uses end-to-end encryption via Cloudflare.

BLE Devices — Pairing Tips

  • BLE devices must be in pairing/advertising mode when you click Connect. Refer to the device manual for how to activate this.
  • EdgeController reconnects automatically to previously paired BLE devices whenever they come into range and enter advertising mode.
  • If a BLE device fails to connect, the system will automatically retry. If it continues to fail, try power cycling the device.
  • Keep the Pi's WiFi connection on 5GHz if possible — 2.4GHz WiFi can interfere with Bluetooth. EdgeController configures this automatically during WiFi setup.

6. The Macro Editor

The macro editor lets you build automated sequences visually. Blocks are connected together to form a flow — when a macro runs, execution passes from block to block following the connections you've drawn.

Creating a Macro

  1. 1Go to Macros in the navigation and click New Macro.
  2. 2Give the macro a name. The editor will open with a blank canvas.
  3. 3Click anywhere on the canvas to add blocks, or use the block panel on the left.
  4. 4Connect blocks by dragging from the output port of one block to the input port of another.
  5. 5Click a block to select it and edit its settings in the panel on the right.
  6. 6Click Save, then Run to execute the macro.
Every macro must start with a Start block. Execution begins there and follows the connections you draw.

Running a Macro

Macros can be started from:

  • The Macros page — click Run next to any saved macro
  • A StreamDeck+ button — assign any macro to a physical button
  • Another macro — use the Run Macro block to chain macros together

Stopping a Macro

Click Stop All in the top bar to immediately halt all running macros. You can also use a Stop All block within a macro to halt everything programmatically.

7. Macro Block Reference

Control Blocks

BlockWhat it does
Start The entry point of every macro. Execution begins here. Every macro needs exactly one Start block.
End Marks the end of a macro path. When execution reaches End, that branch stops.
Stop All Immediately stops all running macros, including the current one.
Run Macro Starts another saved macro. Execution in the current macro continues without waiting.

Timing Blocks

BlockWhat it does
Delay Pauses execution for a set number of seconds before moving to the next block.
Ramp Smoothly transitions a device's intensity from one level to another over a set duration.

Trigger Blocks

BlockWhat it does
Wait: EOM Pauses execution until the EOM arousal sensor reading crosses a set threshold.
Wait: Manual Pauses execution and shows a button in the UI. Execution continues when the user clicks it.
Wait: Paw Pauses execution until a PawPrints button is pressed or the sensor tilts past a set angle. See Section 8.

Logic Blocks

BlockWhat it does
If/Else: EOM Reads the current EOM arousal level. Routes execution to Yes if above threshold, No if below.
Paw: If/Else Checks the current state of a PawPrints button or tilt and routes to Yes or No. See Section 8.
Loop Repeats a section of the macro a set number of times, or indefinitely until stopped.

8. PawPrints Sensor Blocks

The DG-Lab PawPrints (47L120100) is a wireless motion sensor with three buttons. EdgeController can use it as a trigger or decision point inside macros — waiting for a button press or detecting when the sensor has moved past a set position.

Wait: Paw

Pauses the macro until a condition on the PawPrints sensor is met, then lets execution continue.

Button mode

Select Mode: Button and choose which button to wait for (B1, B2, or B3). The macro will pause until that button is pressed, then continue.

Tilt mode

Select Mode: Tilt and choose an axis (X, Y, Z, or Any). The macro will pause until the sensor moves significantly away from its calibrated rest position.

To set the rest position:

  1. Place the sensor in its natural resting position.
  2. Click Calibrate in the block settings — this captures the current reading as the baseline.
  3. Click Set Trip Point to confirm and save the baseline.
  4. Adjust the Sensitivity value — higher values require more movement before triggering.
The tilt trigger fires when the sensor deviates from the calibrated position by more than the sensitivity value in any direction. A sensitivity of 15 works well for moderate movement.

Device selector

If you have more than one PawPrints sensor connected, use the Device dropdown to choose which one this block listens to. If only one is connected, this can be left on automatic.

Paw: If/Else

Takes a snapshot of the sensor's current state and routes execution to one of two paths — Yes or No — without waiting.

The settings are identical to Wait: Paw (button or tilt, axis, calibration, sensitivity, device selector). The difference is that If/Else does not pause — it checks the state at that moment and immediately routes to the appropriate output.

Connections

  • Yes output — button is currently pressed, or sensor is currently outside the calibrated position
  • No output — button not pressed, or sensor is within the calibrated position
Use Wait: Paw to hold the macro until the user is ready. Use Paw: If/Else to branch a macro based on the sensor's current state without stopping execution.

9. Remote Access

After connecting to WiFi for the first time, EdgeController automatically sets up a secure Cloudflare tunnel for your device. No configuration is required.

Web interface

Access the full web interface from anywhere:

https://XXXX.kinkcontrol.org

Replace XXXX with your device ID. The connection is encrypted end-to-end by Cloudflare.

Keep your remote URL private. Anyone with the URL can access your EdgeController web interface. Future versions will include optional password protection.

Finding your device ID

Your device ID is a 4-character code derived from the Pi's hardware serial number. It appears:

  • In the Settings page of the web interface, under Box Settings
  • As part of the WiFi hotspot name during setup (EdgeController-XXXX)
  • In the browser address bar when you access the interface locally

10. Troubleshooting

ProblemSolution
Can't find the EdgeController-XXXX hotspot Wait 90 seconds after powering on. If it still doesn't appear, try power cycling the Pi. The hotspot only appears when the Pi has no internet connection — if it's connected via Ethernet it won't broadcast.
XXXX.local not found in browser Try the Pi's IP address instead (find it in your router's device list). On Windows, install Apple Bonjour for .local support.
BLE device won't connect Make sure the device is in pairing/advertising mode. For PawPrints, hold the button for 3+ seconds until the LED flashes. Try power cycling both the Pi and the device.
Bluetooth drops or is unreliable Move the Pi to a 5GHz WiFi network if possible. 2.4GHz WiFi shares the same radio band as Bluetooth and can cause interference.
Macro stops unexpectedly Check that all devices used in the macro are connected. A device disconnecting mid-macro will cause blocks that reference it to fail. Use the device status indicators in the top bar.
Remote URL not working The tunnel is set up automatically when the Pi first connects to WiFi. If the URL doesn't work, check the tunnel status in Settings. The Pi must have internet access for the tunnel to remain active.
PawPrints tilt not triggering Recalibrate the sensor in its exact resting position and click Set Trip Point. If it's too sensitive or not sensitive enough, adjust the Sensitivity value.
Pi not booting / no lights Check the power supply. The Pi 5 requires a 5V 5A USB-C PSU. Underpowered supplies cause boot failures.
