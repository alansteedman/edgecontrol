Macro Editor

Build automated multi-device sequences using a visual node-based flow editor. Connect blocks together to create timing sequences, ramps, conditional branches, and loops across all your connected devices.

Overview

A macro is a directed graph of blocks connected by wires. Execution starts at the START block and follows wires from one block to the next, running each block's action in order. Blocks can branch (If/Else), loop, or fire in parallel when a non-blocking ramp is followed immediately by a delay.

Non-blocking ramps: Ramp blocks (Coyote Ramp, E-Stim Ramp, Nimble Ramp, Hue Ramp) run in the background by default. This lets you ramp intensity over 60 seconds while simultaneously running delays, triggers, and device changes on the same macro path.

Building a Macro

1. Drag blocks from the left palette onto the canvas.
2. Connect blocks by dragging from an output port (right side of a block) to an input port (left side of another block).
3. Configure blocks by clicking a block — its settings appear in the panel on the right.
4. Every macro must have a START and at least one END block. START has no input, END has no output.
5. Save with the 💾 button, then Run with ▶. Use ■ Stop to abort mid-run.
Tip: Open the 📺 Monitor window for a full-screen live view showing which block is currently executing, countdown timers, and ramp progress bars.

Control Blocks

Every macro starts with START and ends at one or more END blocks. These blocks control macro flow at the top level.

▶
START
The entry point of every macro. Execution begins here. Every macro must have exactly one START block. No configuration required.
■
END
Marks the end of a macro path. When execution reaches END the macro finishes successfully. You can have multiple END blocks in a macro (e.g. at the end of each branch of an If/Else).
⛔
STOP ALL
Immediately zeros all connected devices — sets Coyote intensity to 0, E-Stim power to 0, stops Nimble oscillation. Execution continues to the next block after a brief pause. Use this as an emergency reset within a sequence.
Example — safety reset before a new scene
▶ START→ ⛔ STOP ALL→ ⏱ DELAY 2s→ ⚡ SET Coyote A→ ■ END
▷
SUB MACRO
Runs another saved macro as a sub-routine, then continues. The sub-macro runs to completion (or until the parent is stopped) before the next block fires.
ParameterDescription
MacroWhich saved macro to run
Example — reusable warm-up routine
▶ START→ ▷ SUB MACRO: Warm-up→ ↗ COYOTE RAMP 80→160→ ■ END

Timing Blocks

Timing blocks control pacing within a sequence.

⏱
DELAY
Blocking
Pauses execution for a fixed duration. The macro monitor shows a live countdown. All devices continue running at their current settings during a delay.
ParameterDescription
Duration (s)How long to wait. Minimum 0.5s, no maximum.
Example — hold intensity for 30 seconds
↗ COYOTE RAMP 0→100→ ⏱ DELAY 30s→ ↗ COYOTE RAMP 100→0→ ■ END

Nimble Blocks

Control the NimbleStroker linear actuator — start and stop stroke oscillation, and ramp parameters smoothly over time.

▶
NIMBLE START
Sets all stroke parameters and starts the oscillation. Use this at the beginning of a sequence or after a Nimble Stop to restart with new settings.
ParameterDescription
DeviceWhich Nimble to control (defaults to first connected)
Speed (SPM)Strokes per minute — 6 to 300
Depth (0–1000)Stroke amplitude. 1000 = full range
Nurture (0–200)Adds a gentle vibration texture on top of the stroke
Nature (Hz)Frequency of the nurture vibration — 0.5 to 50 Hz
Example — start a slow, shallow stroke
▶ START→ ▶ NIMBLE START 20 SPM / depth 300→ ⏱ DELAY 60s→ ■ END
⏹
NIMBLE STOP
Stops the stroke oscillation. The device holds its last position. Use before a long delay where you want the device to be still, or at the end of a session.
ParameterDescription
DeviceWhich Nimble to stop (defaults to first connected)
↗
NIMBLE RAMP
Async
Smoothly ramps one stroke parameter from a starting value to a target value over a set duration. Non-blocking by default — the next block fires immediately while the ramp runs in the background.
ParameterDescription
DeviceWhich Nimble to control
ParameterSpeed (SPM), Depth, Nurture, or Nature (Hz)
FromStarting value
ToTarget value
Duration (s)How long the ramp takes
Wait for finishIf checked, next block waits until ramp completes
Example — speed ramp from slow to fast
▶ NIMBLE START 20 SPM→ ↗ NIMBLE RAMP Speed 20→120 / 60s→ ⏱ DELAY 60s→ ⏹ NIMBLE STOP→ ■ END

The ramp and delay run simultaneously — speed climbs to 120 SPM while the macro waits 60 seconds.

Coyote Blocks

Control the DG-Lab Coyote e-stim device — set waveform and intensity instantly or ramp smoothly.

↗
COYOTE RAMP
Async
Smoothly ramps intensity on a Coyote channel from a start value to a target value. Non-blocking by default so you can run delays or device changes in parallel.
ParameterDescription
DeviceSpecific Coyote, or leave blank for all connected Coyotes
ChannelA, B
From (0–200)Starting intensity
To (0–200)Target intensity
Duration (s)Ramp duration
Wait for finishIf checked, macro waits for the ramp to complete before continuing
Example — parallel ramp on both channels
▶ START→ ↗ COYOTE RAMP A: 0→120 / 60s
→ ↗ COYOTE RAMP B: 0→80 / 60s→ ⏱ DELAY 60s ⏸→ ■ END

Connect both ramps to the delay — A and B ramp simultaneously while the delay counts down.

⚡
SET COYOTE (per device/channel)
Instantly sets waveform, intensity, and speed on one channel of a specific Coyote. A separate SET block appears in the palette for each connected Coyote channel (Channel A and Channel B independently).
ParameterDescription
WaveformBuilt-in or custom audio waveform to play on this channel
Intensity (0–200)Output intensity
SpeedWaveform playback speed multiplier (0.25× to 4×)
Example — switch waveform mid-sequence
⚡ SET A: Pulse · 80→ ⏱ DELAY 30s→ ⚡ SET A: Wave · 120→ ■ END

Trigger Blocks

Pause execution until an external condition is met. Great for making macros responsive to sensors and user input.

◈
WAIT: EOM
Blocking
Pauses until the EOM arousal sensor crosses a threshold. Use this to make the macro respond to real-time arousal levels.
ParameterDescription
ConditionArousal > threshold, or Arousal < threshold
Threshold (%)0–100. The arousal level that triggers continuation
Timeout (s)Maximum wait. 0 = wait indefinitely
Example — ramp up until high arousal, then hold
↗ COYOTE RAMP 0→160 / 120s→ ◈ WAIT EOM > 85%→ ⚡ SET A: Pulse · 40→ ■ END
⏸
WAIT: MANUAL
Blocking
Pauses the macro and shows a prompt in the UI. A ▶ Continue banner appears at the top of the macro page. Click it to resume. Useful for multi-stage sessions where a human decision is needed.
ParameterDescription
Prompt textMessage shown on the Continue banner (e.g. "Reposition and click to continue")
🐾
WAIT: PAW
Blocking
Pauses until a PawPrints sensor fires — either a button press or a tilt angle. Use as a hands-free input device.
ParameterDescription
DeviceSpecific PawPrints device, or any
Wait forButton Press or Tilt Angle
ButtonB1, B2, B3, or Any (press mode only)
AxisX, Y, Z, or Any (tilt mode only)
CalibrateSet the reference position for tilt detection
SensitivityHow far the sensor must move to trigger

Logic Blocks

Branch and repeat — build adaptive sequences that respond to sensor readings.

⋈
IF / ELSE
Branch
Checks the current EOM arousal level at this moment and takes the YES or NO output accordingly. Both outputs can lead to different paths that reconverge at an END block.
ParameterDescription
ConditionArousal > or Arousal <
Threshold (%)The level to compare against
Example — high arousal takes a different path
⋈ IF Arousal > 80% YES → ⚡ SET A: 160→ ■ END
NO → ↗ COYOTE RAMP 0→120 / 30s→ ■ END
🐾
PAW: IF / ELSE
Branch
Waits for a PawPrints input, then branches based on which button was pressed or whether the tilt matched the configured axis. YES if the condition matches, NO otherwise.
ParameterDescription
DeviceSpecific PawPrints or any
CheckButton Press or Tilt Angle
ButtonB1, B2, B3, or Any
Example use: Offer the user a choice — B1 = more intensity (YES path), B2/B3 = stop (NO path).
↺
LOOP
Loops back to repeat a section of the macro. The loop body connects from the output of LOOP back to an earlier block, then LOOP's output continues the main sequence when the loop exits.
ParameterDescription
ModeRepeat N times — loops a fixed number of times.
EOM loop — keeps looping while arousal is above or below a threshold.
CountNumber of repetitions (Repeat N times mode)
Condition / ThresholdArousal threshold to exit the loop (EOM mode)
Example — pulse 5 times then ramp
⚡ SET A: Pulse · 80→ ⏱ DELAY 2s→ ⛔ STOP ALL→ ↺ LOOP ×5→ ↗ RAMP 0→160→ ■ END

Worked Example: Warm-up Ramp

Gentle 5-minute warm-up on Coyote, both channels

Starts from zero, slowly builds both channels to a comfortable mid-level over 5 minutes, then holds for another 2 minutes.

1
START — entry point
2
SET Coyote A: Waveform = Breathe, Intensity = 0 — set waveform before ramping
3
SET Coyote B: Waveform = Breathe, Intensity = 0
4
COYOTE RAMP Ch A: 0 → 80, Duration = 300s (non-blocking) — starts ramp in background
5
COYOTE RAMP Ch B: 0 → 60, Duration = 300s (non-blocking) — B slightly lower than A
6
DELAY 300s — holds while both ramps run
7
DELAY 120s — additional hold at target level
8
END
▶ START→ ⚡ SET A:0→ ⚡ SET B:0→ ↗ RAMP A 0→80 / 300s→ ↗ RAMP B 0→60 / 300s→ ⏱ 300s→ ⏱ 120s→ ■ END
